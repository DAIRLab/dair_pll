dair_pll.file_utils

Utility functions for managing saved files for training models.

File system is organized around a “storage” directory associated with data and training runs. The functions herein can be used to return absolute paths of and summary information about the content of this directory.

dair_pll.file_utils.append_by_extension(directory, extension='.pt')[source]

Get absolute path of a new numeric file name for saving.

At various times, we generate a sequence of N files. This method is a helper function that allows a worker to pick a file name to append to the previously generated files.

Example:

# /fldr has contents (0.pt, 1.pt, 2.pt)
append_by_extension("/fldr", ".pt") == '/fldr/3.pt'
Parameters

  • directory (str) – Directory in which file will be saved

  • extension (str) – Extension of desired file name

Return type

str

Returns

Absolute path of new file name

Warning

Doesn’t actually check that previous files are cleanly ordered from 0 upwards. Isn’t thread safe by design; race condition is possible where multiple workers save to the same file name as a result.
